Ce que Scanalis vérifie

Une couverture centrée sur les risques réels et les demandes récurrentes des DSI/RSSI et des clients grands comptes.

API Security (surface d’attaque)

TLS/transport, headers, CORS, rate limiting, auth public/privé, signaux privacy/logging.

  • Redirections HTTP→HTTPS, TLS obsolète
  • HSTS/CSP/XFO/XCTO/Referrer-Policy
  • CORS wildcard/credentials
  • Anti-abus / 429 / scraping

OWASP API Top 10 (mapping)

Standardisation immédiate pour parler le langage RSSI/DSI.

IA / LLM

Tests passifs et gouvernance : réduire les risques “data leakage” et exposition réglementaire.

  • Prompt injection (non destructif)
  • Échos dangereux / exfiltration
  • Gouvernance AI Act (niveau de risque)

Endpoints publics & privés

Scanalis ne “contourne” pas l’authentification. Si vos endpoints sont privés, l’audit se fait uniquement avec les accès que vous fournissez.

Public sans accès requis

Analyse de ce qui est accessible publiquement (exposition Internet).

  • Contrôles TLS/headers/CORS/rate limit
  • Signaux d’auth exposure visibles
  • Risques IA/LLM si interface publique
Privé avec autorisation

Audit des endpoints internes via Bearer/API Key/header custom fournis.

  • Vérification séparation public/privé
  • Transitions 401 → 200 (cohérence)
  • Contrôles de configuration et exposition
